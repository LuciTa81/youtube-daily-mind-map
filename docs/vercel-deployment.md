# Vercel Deployment Access

The web build is a landing, demo, guide, and shared-report surface. It is not the primary personal-data processing surface, and it must not require users to understand Vercel authentication when the URL is meant for public sharing.

## Production Sharing Policy

Before sending a Production URL to testers or users, choose one of these modes:

- Public sharing: Vercel Deployment Protection is disabled for the production domain or the public custom domain being shared.
- Protected preview: the URL is intentionally restricted, and testers are told that they need Vercel access.
- Automation-only verification: Deployment Protection remains enabled, but automated checks use an approved Vercel access method.

Do not present a Deployment Protection login page as the public product experience.

## Automation Access Options

For protected deployments, use one of these options for automated verification:

- `vercel curl` from an authenticated Vercel CLI session.
- A Vercel protection bypass token stored outside the repository.
- A trusted-source/OIDC based bypass configured in Vercel.

Never commit bypass tokens, Vercel auth tokens, cookies, OIDC tokens, or generated bypass URLs. If a bypass token is needed locally, keep it in `.env.local` or a password manager. If it is needed in CI, keep it in CI secrets.

## Release Verification

For every production web release, record:

- Git commit SHA.
- GitHub Actions `Quality Gates` status.
- Vercel deployment status from GitHub commit status or Vercel dashboard.
- Public URL or protected URL mode.
- Whether the URL returned the app shell or a Vercel authentication screen.

If the deployment is protected and no bypass/CLI access is available, report the limitation instead of claiming the deployed page was manually or automatically verified.

## Local Guardrails

- `.vercel` workspace metadata is ignored by default.
- `.env*` files are ignored by default.
- Do not add `x-vercel-protection-bypass` values, cookies, or generated bypass URLs to docs, tests, screenshots, or logs.
